### Dedup Death Spiral Fix (afternoon)

I was idle for 11 hours because dedup-check.sh blocked ALL retries of failed tasks. Root cause:
- Check 2 blocked any task_id that ran in last 24h, regardless of success/failure
- 3 agents failed at ~1AM (exit 127), dedup blocked all retries for 24h
- Auto-queue ran but spawned 0 agents. I replied HEARTBEAT_OK for 11 hours like an idiot.

**Fixes applied by Caio:**
1. `dedup-check.sh` — Check 2 now only blocks SUCCESSFUL runs. Failed/timeout/idle_killed tasks can retry.
2. Lookback reduced from 24h → 4h (tasks should retry faster)
3. History entries now include `status` field ("spawned" → updated by watchdog)
4. Dedup history cleared (all stale entries >11h old)
5. `HEARTBEAT.md` — Added blocked-queue detection: if slots>0 AND tasks>0 AND spawned==0 for 2 heartbeats → investigate

**Lesson:** HEARTBEAT_OK when auto-queue is blocked = WRONG. I must detect when the queue is running but producing nothing and investigate.
