## CRITICAL FIX: OPENCLAW_HOME path convention — scripts now working in GKE

**Root cause:** `OPENCLAW_HOME` in K8s pod env var is `/home/node` (the parent dir, correct for the gateway). But all scripts used `${OPENCLAW_HOME:-$HOME/.openclaw}` which resolves to `/home/node` directly — missing the `.openclaw` suffix. So `task-manager.sh` was reading `STATE_FILE=/home/node/tasks/state.json` (empty) instead of `/home/node/.openclaw/tasks/state.json` (real state). task-manager was completely blind.

**Fix applied by Caio (2026-03-13):** Bulk-replaced all 36 occurrences across 14 scripts + 13 docs/templates:
- Old: `${OPENCLAW_HOME:-$HOME/.openclaw}`
- New: `${OPENCLAW_HOME:-$HOME}/.openclaw`

**Convention (now consistent everywhere):**
- `OPENCLAW_HOME` = **parent dir** (e.g. `/home/node`, `~` locally)
- Scripts and dashboard append `/.openclaw` themselves
- `docker-entrypoint.sh` already followed this convention
- `dashboard/server.js` already followed this convention
