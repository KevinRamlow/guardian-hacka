## Architecture Refactor: Atomic Architecture (v4)

**Major consolidation** — went from 13+ scripts to 4 scripts + 1 brain.

**New architecture:**
| Component | Sole Responsibility |
|---|---|
| `task-manager.sh` | State CRUD + transitions (flock-protected) |
| `dispatcher.sh` | THE only spawn path: Linear + state + spawn + exit-code watcher |
| `kill-agent-tree.sh` | Kill PID tree (utility) |
| `guardrails.sh` | Invariant checks |
| **HEARTBEAT.md** | The brain: Slack reporting, timeouts, orphans, auto-queue, callbacks |

**What changed:**
- `dispatcher.sh` completely rewritten — self-contained, no spawn-agent.sh dependency
- Exit-code watcher baked into dispatcher (bg process polls agent PID, auto-transitions on death)
- Linear task creation is MANDATORY (not optional) — either `--title` for new or `--task` for existing
- `HEARTBEAT.md` is the sole monitor/reporter — no supervisor, no reporter script
- Streaming set to `block` (buffers full response before sending to Slack)
- `lightContext: true` saves ~30% tokens per heartbeat
- WhatsApp integration removed completely from openclaw.json

**Deprecated scripts (archived to .archive/):**
supervisor.sh, spawn-agent.sh, reporter.sh, link-logs-to-linear.sh, alert-dedup.sh, agent-logger.sh, dedup-check.sh, dispatch-guard.sh, interactive-checkpoint.sh

**How completions work now:**
1. dispatcher.sh spawns agent + launches exit-code watcher (background process)
2. When agent dies, watcher: checks output quality → transitions state → logs to Linear
3. Heartbeat (5min) reads state.json → reports to Slack → handles timeouts/orphans/callbacks

**docker-entrypoint.sh updated:** Removed supervisor.sh background loop (heartbeat handles everything).
