| Component | Sole Responsibility |

### Agentless Eval Dispatch Feature (2026-03-10 ~22:00 UTC)
**New feature:** `dispatcher.sh --eval` launches eval processes directly without spawning an agent. Agents were wasting tokens looping and checking on eval progress — now evals run as bare processes.

**Changes:**
- `dispatcher.sh`: Added `--eval`, `--parent`, `--eval-config`, `--eval-dataset`, `--eval-workers` flags
- `task-manager.sh`: Added `todo → eval_running` as valid transition, `parentTask` field in state schema
- `dashboard/server.js`: Shows eval_running and callback_pending tasks in active list with parent linkage
- `HEARTBEAT.md`: Updated callback handling to pass parent context to callback agents
- `SOUL.md`: Documented agentless eval dispatch, updated rules

**Usage:**
```bash
bash scripts/dispatcher.sh --eval --title "Eval: post GUA-1101" --parent AUTO-XX
bash scripts/dispatcher.sh --eval --title "Eval: custom" --eval-config path.yaml --eval-workers 10
```

**Flow:** dispatcher creates Linear task → launches eval process → state=eval_running → watcher detects death → extracts accuracy from metrics.json → state=callback_pending → heartbeat spawns callback agent with full context

**Key improvement:** No agent tokens wasted on eval babysitting. Eval watcher auto-extracts accuracy and stores in history[]. Default timeout: 90min.
