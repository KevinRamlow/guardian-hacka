### Supervisor Duplicate Callback Bug (CRITICAL)
- Supervisor triggers callback_pending → dispatch_callback EVERY 30s
- `dispatch_callback()` transitions to agent_running and spawns callback agent
- But callback goes to MAIN session, not sub-agent
- Main session processes it, marks done
- Meanwhile supervisor re-detects callback_pending (race condition) and re-dispatches
- Result: 4-5 duplicate callbacks per task completion
- **Fix needed:** Track dispatched callbacks (e.g., `callbackDispatched: true` flag) or transition immediately after dispatch

### Key Learnings
- Guardian evals need ~40-45min for 121 cases with 10 workers (30min too short)
- Langfuse telemetry must be disabled for evals (`enable_langfuse=False` in wire())
- Dashboard SSE connections need explicit cleanup on rapid navigation
- Session file matching by task ID content is more reliable than time-based matching

### ⚠️ LESSON: When Removing HTML Sections, ALWAYS Remove JS References Too
**What happened:** Removed "Source Breakdown" and "Processes" HTML sections but forgot to remove `renderSourceBreakdown()` and `renderProcesses()` calls from `render()`. Those functions tried to access `$('#sourceBreakdown')` and `$('#processesContainer')` which no longer existed → JS error → **blocked ALL subsequent render functions** (eval, recent completions, tokens).

**Rule for dashboard edits:**
1. When deleting an HTML section, grep for ALL references to its ID in the JS
2. Remove the render function call from `render()`
3. Remove the render function itself
4. Test by checking browser console for errors (or `node -e` to parse the script)
5. **Never commit HTML removal without also cleaning JS references in the same commit**
