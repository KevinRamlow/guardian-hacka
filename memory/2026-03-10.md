## CRITICAL FIX: Duplicate Completion Alerts (Done Bug)

**Root cause found:** I was reporting completed tasks every heartbeat because HEARTBEAT.md didn't tell me to check `reportedAt` before posting. REVIEW-317 got spammed 20+ times to Caio's DM.

**Fix applied to HEARTBEAT.md:**
- MUST check `reportedAt` field in state.json before reporting any completion or failure
- If `reportedAt` is set → task already reported → SKIP IT
- After reporting, ALWAYS run: `bash scripts/task-manager.sh set-field <TASK_ID> reportedAt "$(date -u +%Y-%m-%dT%H:%M:%SZ)"`
- Added to "What NOT to do" section as well

**Rule:** Never report the same task twice. `reportedAt` is the dedup guard — check it BEFORE posting, set it AFTER posting.
