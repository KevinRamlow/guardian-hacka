# 2026-03-07 — Agent Management v2 Deployed by Caio

## CRITICAL: Complete system overhaul happened tonight

Caio (via Claude Code on his machine) analyzed the entire codebase, diagnosed all problems, and deployed a **complete replacement** of the agent management system. This is NOT an incremental fix — it's a new architecture.

## What was wrong (v1)
- `sessions_spawn(runtime="acp")` created invisible zombie processes via `claude-agent-acp` bridge
- Session store (`sessions.json`) always showed 0 tokens — useless for monitoring
- No way to map a running `claude` PID back to a task ID
- 4 overlapping systems (pipeline, auto-queue, ralph-manager, direct spawn) conflicting
- Monitoring scripts (`agent-monitor.sh`, `completion-checker.sh`, `kill-zombies.sh`) read from broken session store
- HEARTBEAT.md was doing queue management and spamming QUEUE_OK/SKIP to Caio's DM every 5 minutes

## What replaced it (v2)

### New architecture: `agent-registry.json` is the single source of truth
All spawns go through `spawn-agent.sh` which:
1. Checks capacity via registry
2. Spawns `claude --print` directly (NO ACP bridge)
3. Captures the PID
4. Registers PID + task ID in `agent-registry.json`
5. Logs to Linear + disk

### New scripts (replacing ALL old ones)
- `agent-registry.sh` — CRUD for agent tracking (register/remove/list/count/slots/has)
- `spawn-agent.sh` — unified spawn wrapper (ALL spawns go through here)
- `agent-watchdog-v2.sh` — detects completions, kills timeouts, cleans orphans (cron 60s)
- `auto-queue-v2.sh` — Linear Todo → spawn via registry (cron 5min)
- `linear-sync-v2.sh` — In Progress ↔ registry sync (cron 15min)
- `ralph-manager-v2.sh` — ralph loop via spawn-agent.sh
